native validation does not have to just happen on submit. we can trigger the browser's validation run directly via JavaScript - here, we do it when a user leaves an input, to provide instant error messages if something wasn't right with the data they just entered. highly annoying and not user-friendly (user is forced to enter valid data before being able to continue with anything else on the page), but demonstrates the point.

BUG: entering something other than a number won't trigger any error in Opera
